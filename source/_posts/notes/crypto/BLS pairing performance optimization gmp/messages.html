Order-Fair Consensus in the Permissionless Setting : https://eprint.iacr.org/2021/139.pdf

This paper presents an order-fairness property for the permissionless setting. In the permissioned setting (https://eprint.iacr.org/2020/269, CRYPTO 2020), they define order-fairness in terms of transactions being received by a large fraction of nodes. In the permissionless setting, the ordering is determined by looking at the time when the transaction was initially propagated through the network. This work seems relevant to the discussion earlier to minimize MEV.

20:40
None of the optimizations are Nim specific.

Nim does make certain things easier: Go has no support for generics for example so you need one code per curve, Rust has restricted const generics/integer values as types so it's harder to use compile-time bounds.

Both need a double pass to generate the assembly then the code that uses it while everything is one shot in Nim, the assembler is also clearer and maps almost 1-1 to paper algorithms. (though Rust procedural macros might solve the 2-pass issue?)

Adding BW6-761 only required adding the parameters in a config file and then configuring the pairing function (which I'm stuck at).
20:44
Also I can parse hex numbers at compile-time, do Montgomery domain conversion in the compiler and also optimize dispatch when certain values are known to be zero, 1 or -1 at compile-time without hardcoding that per curve (useful for Frobenius maps and multiplication by non-residue).
