Right, the question is about whether there's a more efficient way to do it than just a 64×64 bit multiplication and then a 128/64 bit division. There definitely isn't on architectures where those are operations directly supported by the hardware
20:45

20:53
Note: divq causes a #DE exception if the result does not fit in 64 bits; see the Intel 64 and IA-32 Architectures Software Developer’s Manual Volume 2A page 3-285. This might make it difficult for some compilers to use it.
