Daira Hopwood
If a non-interactive argument of knowledge is constructed by Fiat–Shamir in the recommended way, which includes an encoding of the instance at the start of the hashed transcript, then making any change to the instance will change all of the verifier's challenges. And so, if the proof system is also zk, there is (heuristically) no reason to believe that a zk-NARK proof with the old instance will be useful to generate any new instance with a related witness.

It *might* be possible to make this heuristic argument more rigorous, but I haven't seen that written out in any paper.
21:52
This doesn't apply to the case of creating a new proof for the same instance. You can definitely do that for Groth16.
