Omer
We started exploring with formal verification tools couple of years ago where the original idea was to see how useful it would be to have a higher level language for expressing MPC protocols that can automatically be mapped to runnable (Rust) code. The main purpose is to minimize the manual work needed to implement a paper protocol and to reduce errors in this step: operations and constructs can be added to the language as needed, but would ideally only have to be analysed once. The secondary purpose is to enable formal analysis by mapping to a formal model such as ProVerif or other model checkers instead.

To use TLA+ for cryptography its a bit of a challenge. But for distributed system it is perfect. At the early stages of white-city we tested TLA+ and coq (https://github.com/KZen-networks/white-city/tree/master/RelayProofsOfConcept/Formal-spec/coq). For the model of MPC with relay server. We were very happy with the results - by building with TLA+ you are required to be very accurate about your spec. Later when you run it you see clearly where the spec is violated (invariants are not maintained)

verification: have used both,

Verifpal is more approachable (can be learnt in 1h), but less expressive and has missed some attacks in earlier versions; it now has ProVerif and Coq translators, which is really cool, see recent talk at https://verifpal.com/res/pdf/quarkslab2020.pdf
